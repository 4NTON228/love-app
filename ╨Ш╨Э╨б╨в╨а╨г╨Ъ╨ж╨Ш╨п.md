## 1. Концепт дизайна

- **Основные цвета:**

## 4. Настройка Supabase (ШАГ 1)

### 4.1. Создай аккаунт
1. Зайди на https://supabase.com
2. Нажми "Start your project"
3. Войди через GitHub (или email)
4. Нажми "New project"
5. Заполни:
   - Name: `love-app`
   - Database Password: придумай сложный пароль (ЗАПИШИ!)
   - Region: выбери ближайший (например, Frankfurt)
6. Нажми "Create new project" и подожди 2 минуты

### 4.2. Запиши ключи
1. Зайди в Settings → API
2. Запиши:
   - **Project URL** — что-то вроде `https://xxxxx.supabase.co`
   - **anon public key** — длинная строка

### 4.3. Создай таблицы
Зайди в **SQL Editor** (левое меню) и выполни этот SQL:

```sql
-- ================================================
-- SQL-СХЕМА ДЛЯ LOVE APP
-- Копируй целиком и вставь в SQL Editor в Supabase
-- ================================================

-- 1. Таблица профилей пары
CREATE TABLE profiles (
  id UUID REFERENCES auth.users(id) PRIMARY KEY,
  name TEXT NOT NULL,
  avatar_url TEXT,
  partner_id UUID REFERENCES auth.users(id),
  couple_start_date DATE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 2. Таблица событий календаря
CREATE TABLE calendar_events (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) NOT NULL,
  title TEXT NOT NULL,
  description TEXT,
  event_date DATE NOT NULL,
  photo_url TEXT,
  emoji TEXT DEFAULT '❤️',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 3. Таблица моментов (воспоминаний)
CREATE TABLE moments (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) NOT NULL,
  title TEXT NOT NULL,
  description TEXT,
  photo_url TEXT,
  mood TEXT DEFAULT 'love',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 4. Таблица планов
CREATE TABLE plans (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) NOT NULL,
  title TEXT NOT NULL,
  completed BOOLEAN DEFAULT FALSE,
  category TEXT DEFAULT 'dream',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 5. Таблица настроек пары
CREATE TABLE couple_settings (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) NOT NULL,
  love_message TEXT DEFAULT 'Я тебя люблю ❤️',
  next_meeting TIMESTAMPTZ,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 6. Создаём Storage bucket для фото
INSERT INTO storage.buckets (id, name, public)
VALUES ('photos', 'photos', true);
```

### 4.4. Настрой Row Level Security (RLS)

Выполни в SQL Editor:

```sql
-- ================================================
-- ПРАВИЛА БЕЗОПАСНОСТИ (Row Level Security)
-- Только авторизованные пользователи видят данные
-- ================================================

-- Включаем RLS на всех таблицах
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE calendar_events ENABLE ROW LEVEL SECURITY;
ALTER TABLE moments ENABLE ROW LEVEL SECURITY;
ALTER TABLE plans ENABLE ROW LEVEL SECURITY;
ALTER TABLE couple_settings ENABLE ROW LEVEL SECURITY;

-- Profiles: видишь и редактируешь только свой профиль,
-- но можешь видеть профиль партнёра
CREATE POLICY "Users can view own and partner profile"
  ON profiles FOR SELECT
  USING (
    auth.uid() = id
    OR id = (SELECT partner_id FROM profiles WHERE id = auth.uid())
  );

CREATE POLICY "Users can update own profile"
  ON profiles FOR UPDATE
  USING (auth.uid() = id);

CREATE POLICY "Users can insert own profile"
  ON profiles FOR INSERT
  WITH CHECK (auth.uid() = id);

-- Calendar: оба партнёра видят все события
CREATE POLICY "Couple can view calendar"
  ON calendar_events FOR SELECT
  USING (
    auth.uid() = user_id
    OR user_id = (SELECT partner_id FROM profiles WHERE id = auth.uid())
  );

CREATE POLICY "Users can insert calendar events"
  ON calendar_events FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update own calendar events"
  ON calendar_events FOR UPDATE
  USING (auth.uid() = user_id);

CREATE POLICY "Users can delete own calendar events"
  ON calendar_events FOR DELETE
  USING (auth.uid() = user_id);

-- Moments: оба партнёра видят все моменты
CREATE POLICY "Couple can view moments"
  ON moments FOR SELECT
  USING (
    auth.uid() = user_id
    OR user_id = (SELECT partner_id FROM profiles WHERE id = auth.uid())
  );

CREATE POLICY "Users can insert moments"
  ON moments FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update own moments"
  ON moments FOR UPDATE
  USING (auth.uid() = user_id);

CREATE POLICY "Users can delete own moments"
  ON moments FOR DELETE
  USING (auth.uid() = user_id);

-- Plans: оба партнёра видят и могут отмечать выполненными
CREATE POLICY "Couple can view plans"
  ON plans FOR SELECT
  USING (
    auth.uid() = user_id
    OR user_id = (SELECT partner_id FROM profiles WHERE id = auth.uid())
  );

CREATE POLICY "Users can insert plans"
  ON plans FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Couple can update plans"
  ON plans FOR UPDATE
  USING (
    auth.uid() = user_id
    OR user_id = (SELECT partner_id FROM profiles WHERE id = auth.uid())
  );

CREATE POLICY "Users can delete own plans"
  ON plans FOR DELETE
  USING (auth.uid() = user_id);

-- Couple settings
CREATE POLICY "Couple can view settings"
  ON couple_settings FOR SELECT
  USING (
    auth.uid() = user_id
    OR user_id = (SELECT partner_id FROM profiles WHERE id = auth.uid())
  );

CREATE POLICY "Users can insert settings"
  ON couple_settings FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Couple can update settings"
  ON couple_settings FOR UPDATE
  USING (
    auth.uid() = user_id
    OR user_id = (SELECT partner_id FROM profiles WHERE id = auth.uid())
  );

-- Storage: авторизованные пользователи могут загружать фото
CREATE POLICY "Authenticated users can upload photos"
  ON storage.objects FOR INSERT
  WITH CHECK (bucket_id = 'photos' AND auth.role() = 'authenticated');

CREATE POLICY "Anyone can view photos"
  ON storage.objects FOR SELECT
  USING (bucket_id = 'photos');
```

### 4.5. Создай два аккаунта

1. Зайди в **Authentication** → **Users**
2. Нажми **"Add user"** → **"Create new user"**
3. Создай:
   - Email: `anton@love.app` / пароль: (твой пароль)
   - Email: `девушка@love.app` / пароль: (её пароль)
4. ⚠️ Поставь галочку **"Auto Confirm User"** для обоих!

### 4.6. Свяжи аккаунты как пару

После создания пользователей, скопируй их UUID из таблицы Users.
Потом выполни в SQL Editor (подставь реальные UUID):

```sql
-- Замени UUID на реальные из Authentication → Users
INSERT INTO profiles (id, name, partner_id, couple_start_date)
VALUES
  ('UUID_АНТОНА', 'Антон', 'UUID_ДЕВУШКИ', '2024-01-15'),
  ('UUID_ДЕВУШКИ', 'Имя девушки', 'UUID_АНТОНА', '2024-01-15');

-- Добавь начальное любовное сообщение
INSERT INTO couple_settings (user_id, love_message, next_meeting)
VALUES ('UUID_АНТОНА', 'Ты — лучшее, что случилось в моей жизни ❤️', NULL);
```

## 5. Создание фронтенда (ШАГ 2)

### 5.1. Установи инструменты
1. Установи **Node.js** — https://nodejs.org (скачай LTS версию)
2. Установи **Git** — https://git-scm.com
3. Установи **VS Code** — https://code.visualstudio.com

### 5.2. Создай проект
Открой терминал (cmd или PowerShell) и выполни:

```bash
npm create vite@latest love-app -- --template react
cd love-app
npm install
npm install @supabase/supabase-js react-router-dom lucide-react date-fns
```

### 5.3. Скопируй все файлы из этого архива в проект
(Все файлы приложены ниже в этом пакете)

### 5.4. Создай файл `.env` в корне проекта:
```
VITE_SUPABASE_URL=https://твой-проект.supabase.co
VITE_SUPABASE_ANON_KEY=твой-anon-ключ
```

### 5.5. Запусти локально:
```bash
npm run dev
```
Открой http://localhost:5173 в браузере.

## 6. Деплой на Netlify (ШАГ 3)

### 6.1. Подготовка
1. Создай аккаунт на https://github.com
2. Создай новый репозиторий `love-app`
3. В терминале:
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЮЗЕРНЕЙМ/love-app.git
git push -u origin main
```

### 6.2. Деплой
1. Зайди на https://netlify.com
2. Войди через GitHub
3. Нажми "Add new site" → "Import an existing project"
4. Выбери репозиторий `love-app`
5. Настройки сборки:
   - Build command: `npm run build`
   - Publish directory: `dist`
6. В **Environment variables** добавь:
   - `VITE_SUPABASE_URL` = твой URL
   - `VITE_SUPABASE_ANON_KEY` = твой ключ
7. Нажми **Deploy site**

### 6.3. Настрой домен (опционально)
- В Netlify: Site settings → Domain management
- Можешь использовать бесплатный домен `*.netlify.app`
- Или купить свой домен (например, `our-love.app`)

## 7. Установка на iPhone (ШАГ 4)

### 7.1. Как установить PWA на iPhone:
1. Открой Safari на iPhone
2. Зайди на адрес твоего сайта (например `https://love-app.netlify.app`)
3. Нажми кнопку "Поделиться" (квадрат со стрелкой вверх)
4. Прокрути вниз и нажми **"На экран Домой"**
5. Введи название (например "💕 Love")
6. Нажми "Добавить"
7. На рабочем столе появится иконка приложения!

### 7.2. Важно для iPhone:
- PWA на iOS работает ТОЛЬКО через Safari (не Chrome!)
- Приложение будет работать в полноэкранном режиме
- Данные сохраняются между сессиями
- Push-уведомления на iOS доступны с iOS 16.4+
